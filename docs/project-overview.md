# Project Overview

This repository documents a defensive homelab portfolio built to practice Linux administration, Docker operations, network documentation, service hardening, backups, logging, and incident response thinking.

The goal is to show how a small self-hosted environment can be planned, operated, reviewed, and improved with a Blue Team mindset.

## Scope

This project includes documentation for:

- Homelab goals and architecture.
- Hardware and operating assumptions.
- Docker-based service planning.
- Network layout using safe placeholders.
- Security hardening practices.
- Logging, monitoring, backups, and incident response notes.
- Bitcoin and Lightning node operational security lessons.
- Media stack security considerations.

## Safety and Privacy Rules

This repository must not include:

- Real public or private IP addresses.
- Credentials, secrets, recovery phrases, API keys, or tokens.
- Private hostnames or personally identifying network details.
- Screenshots that reveal internal services, usernames, domains, or addresses.
- Sensitive home network configuration.

Use placeholders such as `<LAN_SUBNET>`, `<HOSTNAME>`, `<SERVICE_PORT>`, and `<REDACTED>` when examples are needed.

## Portfolio Value

This project is intended to demonstrate practical defensive skills, including:

- Clear technical documentation.
- Risk-aware service planning.
- Secure defaults and least privilege thinking.
- Log review and monitoring habits.
- Backup and recovery planning.
- Incident response note-taking.
